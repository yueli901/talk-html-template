Section 1

Overview

What this template gives you

  • Pure static — hosts free on GitHub Pages
  • Works offline — installable, no internet after first load
  • 1-D navigation — any arrow steps the talk
  • Beamer-style chrome — section headline + footer
  • Speaker view with notes & timer
  • Offline math via KaTeX
  • Draw on slides (chalkboard)
  • Grouped overview — scroll to browse
  • Floating PDF download + QR
  • Narration playback from object storage

The bar at the top shows the section; the footer shows presenter, title, date and page.

Section 2

Navigating

Navigating (1-D / linear)

Every direction moves one step:

  • → ↓ Space — next
  • ← ↑ — previous
  • Esc or the grid button — the map view

Presenting:

  • S — speaker view (notes + next + timer)
  • F fullscreen · B black · C pen
  • Alt+click — zoom a figure

The map view (press Esc)

  • all slides are laid out as a map, grouped by section (one column per section)
  • drag to pan · scroll / pinch to zoom — smoothly, like a map
  • click a slide to jump there; Esc again to return

Try it: press Esc now, pan and zoom, then click a slide.

Section 5

Share & offline

Narration playback

Pre-record audio, host it on object storage, list it in a manifest — the page plays it and auto-advances.

  • the ▶ Play narrated button appears only if audio exists
  • one clip per slide → re-record any slide independently
  • the page only loads and plays — no recording

Demo ships 3 clips — press ▶ Play narrated.


// narration/manifest.json
{
  "base": "https://cdn.example.com/talk/",
  "clips": { "intro":"s00.m4a", "overview":"s01.m4a", "nav":"s02.m4a" }
}

keys match each slide's data-audio. Bucket must allow CORS.

Section 6

Deploy

Reuse this template

  1. Click Use this template on GitHub → new repo
  2. Set window.TALK (top of index.html) and edit the <section> slides
  3. Group slides into sections with <section data-section="…">
  4. Enable Settings → Pages → main / root
  5. (optional) add narration clips + narration/manifest.json

Full instructions in README.md.
